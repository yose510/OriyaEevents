# 🎉 OriyaEvents - מערכת הזמנות דיגיטליות

## 📦 מה יש בחבילה:

✅ **3 קבצים בלבד!**
- `index.html` - דף ראשי + התחברות/הרשמה
- `dashboard.html` - לוח בקרה ללקוחות
- `admin.html` - פאנל ניהול למנהל
- `database-schema.sql` - SQL ליצירת טבלאות

---

## 🚀 הוראות התקנה (10 דקות)

### שלב 1: הגדרת Supabase (5 דקות)

1. **עבור ל-Supabase SQL Editor:**
   ```
   https://supabase.com/dashboard/project/kzxyewemuykimsblotif/sql/new
   ```

2. **העתק והדבק את כל התוכן מ-`database-schema.sql`**

3. **לחץ RUN (Ctrl+Enter)**

4. **אמת שהטבלאות נוצרו:**
   - Table Editor → צריכות להופיע 4 טבלאות:
     - users
     - invitations
     - invitations_sent
     - payments

5. **הגדר URL Redirect:**
   - Settings → Auth → URL Configuration
   - **Site URL:** `https://yose510.github.io/OriyaEvents/`
   - **Redirect URLs:** `https://yose510.github.io/OriyaEvents/**`
   - שמור!

---

### שלב 2: העלאה ל-GitHub (5 דקות)

2. **מחק את הקבצים הישנים** (אם יש)

### שלב 3: יצירת משתמש אדמין (2 דקות)

1. **עבור לאתר:**
   ```
   https://yose510.github.io/OriyaEvents/
   ```

2. **הרשם עם כל אימייל שתרצה**
   - לדוגמה: `admin@oriya.com`
   - סיסמה: מה שתרצה (לפחות 6 תווים)

3. **אמת את האימייל** (בדוק תיבת דואר)

4. **הפוך לאדמין ב-Supabase:**
   - Table Editor → `users`
   - מצא את המשתמש שיצרת
   - שנה את `is_admin` מ-`false` ל-`true`
   - שמור

5. **התחבר מחדש** - תועבר ישר לפאנל אדמין! 🎉

## 💡 איך המערכת עובדת:

### למשתמשים רגילים:
1. נרשמים באתר
2. מקבלים 0 הזמנות (צריך לקנות חבילה)
3. יוצרים הזמנות
4. מצלמים ביט על ₪300
5. אדמין מאשר → מקבלים 300 הזמנות

1. מתחבר לפאנל אדמין
2. רואה את כל המשתמשים
3. לוקח לחיצה על "ערוך"
4. משנה "הזמנות בחבילה" ל-300
5. שולם? → כן
6. שמור → הלקוח מקבל את החבילה!

## 📊 מבנה Database:

```sql
users (משתמשים)
├── id
├── email
├── full_name
├── package_invites (כמה בחבילה)
├── package_sent (כמה נשלחו)
├── package_paid (שולם?)
└── is_admin (מנהל?)

invitations (הזמנות)
├── id
├── user_id
├── groom_name
├── bride_name
├── event_date
├── event_time
├── venue
└── address

invitations_sent (מעקב)
├── id
├── user_id
├── guest_name
└── guest_phone

payments (תשלומים)
├── id
├── user_id
├── amount
├── package_invites
└── status
```
